02.00_MODELOS
==========

02.10_MODELO_PRINCIPAL
- Usar para: arquitectura, refactor multi-file, debugging difícil, decisiones.
- Parámetros sugeridos:
  - temperature 0.2–0.4
  - max_tokens según tarea (ver 03.xx)

02.20_MODELO_ECONOMICO
- Usar para: navegación, resúmenes mecánicos, cost report, boilerplate simple.
- Parámetros sugeridos:
  - temperature 0.2–0.5
  - max_tokens menor

==========
03.00_MATRIZ_ROUTING
==========

03.10_PLAN_ARCH
- modelo: principal
- contexto: L0 -> L1
- salida: plan + riesgos + estimate + checklist
- apply: PLAN_ONLY

03.20_NAV_FIND
- modelo: económico
- contexto: L0 (RADAR_INDEX)
- salida: paths exactos + recomendación de next open
- apply: PLAN_ONLY

03.30_DOC_CANON
- modelo: económico para doc mecánica; principal para doctrina
- contexto: L0/L1
- salida: secciones completas (FILE_CONTENT)
- apply: PLAN_ONLY (salvo que sea auto-doc en artifacts)

03.40_CODEGEN
- modelo: principal si afecta core; económico si boilerplate
- contexto: L1 (snippets targets)
- salida: patch + archivos nuevos
- apply: APPLY_APPROVED

03.50_REFACTOR
- modelo: principal
- contexto: L1 -> L2
- salida: patch + plan de validación + rollback plan
- apply: APPLY_APPROVED

03.60_DEBUG_TROUBLE
- modelo: principal
- contexto: L1 (logs + snippet)
- salida: hipótesis única + cambio mínimo + prueba (anti-shotgun)
- apply: APPLY_APPROVED

03.70_COST_REPORT
- modelo: económico
- contexto: L0 (logs)
- salida: report (por task/proj)
- apply: PLAN_ONLY

03.80_RADAR_REVIEW
- modelo: económico
- contexto: L0 (INDEX) o L2 (CORE selectivo)
- salida: anomalías, duplicados, archivos fuera de naming
- apply: PLAN_ONLY

==========
04.00_ESCALAMIENTO_Y_REDUCCION_DE_CONTEXTO
==========

04.10_ESCALAMIENTO
- Iniciar L0/L1 por defecto.
- Subir a L2 solo si:
  - la tarea es REFACTOR
  - o falló por falta de contexto (y se documenta)
- L3 solo auditoría puntual.

04.20_REDUCCION
- Si el costo sube:
  - volver a L1
  - chunk de CORE por módulo
  - usar modelo económico para NAV/DOC mecánica.

==========
05.00_APPLY_VS_PLAN_ONLY
==========

05.10_DEFAULT
- Todo parte PLAN_ONLY.

05.20_APPLY
- Apply solo con gate humano + checkpoint + preflight OK.

==========
06.00_FALLBACK_(MVP_MANUAL)
==========

- Si el modelo principal falla:
  - registrar TROUBLE
  - humano decide:
    - reintentar con más contexto
    - o pasar a económico (si tarea lo permite)
- No fallback automático en MVP.

==========
07.00_ANTIPATRONES
==========

- FULL (L3) por defecto.
- Modelo caro para NAV_FIND.
- Apply sin plan.
- Más de 2 modelos en MVP.
- Mezclar routing policies duplicadas.
