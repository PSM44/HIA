01.00_INTRO
01.01 Propósito
Este documento describe un caso real (no solo teórico) para aprender HIA ejecutándolo:
- Se construye una matriz de sensibilidad con cabeceras Precio y Absorción.
- El valor dentro de la matriz es TIR (Tasa Interna de Retorno).
- Se documenta el proceso completo: decisiones de stack, roles IA, handoffs, validación, evidencia y aprendizaje.

01.02 Qué significa “caso real” aquí
- Todo cambio se hace en el repo, con evidencia (RADAR, tags, STATE.HISTORY).
- La IA no “hace magia”: USER ejecuta comandos, revisa diffs, aprueba APPLY.
- El resultado final debe ser reproducible desde cero en otra máquina.

01.03 Abreviaciones (definidas 1 vez)
- HIA: Human+IA Architecture (framework operativo de gobernanza).
- TL.BOOTSTRAP: Tooling Layer Bootstrap (kit mínimo para iniciar proyecto y operar con evidencia).
- IDE: Integrated Development Environment (VS Code).
- IA_ORCH: IA Orquestador (rol que coordina).
- IA_CLOUD_WEB: IA cloud en navegador (orquestación y planificación).
- IA_CLOUD_IDE: IA en VS Code (Continue) para revisar/editar.
- IA_LOCAL: IA local (Ollama) para tareas offline/sensibles.
- PLAN_ONLY: modo donde la IA solo planifica (no aplica cambios).
- APPLY: ejecución de cambios en archivos.
- VALIDATE: verificación (tests/scripts/sanity checks).
- LOG: registro/evidencia (STATE.HISTORY + commit/tag + RADAR).
- CP.MIN: Context Pack mínimo para pasar contexto a una IA.
- CHP: Change Pack (resumen de cambios).
- EP: Evidence Pack (evidencia de validación).

07.00_DEFINICIÓN DEL MVP MATEMÁTICO (PARA EVITAR TRAMPA)
07.01 Modelo MVP recomendado (simple y verificable)
- Asumir flujos mensuales:
  - t0: inversión inicial negativa (por ejemplo costos totales o CAPEX simplificado)
  - t1..tn: ingresos netos (dependen de precio * unidades vendidas, con absorción mensual)
- TIR (IRR) calculada sobre la serie de flujos.
Nota: la TIR real de un proyecto inmobiliario puede requerir más detalle; aquí buscamos MVP reproducible.

07.02 Variables del sensitivity
- Precio: vector de precios (ej: -10%, base, +10% o valores discretos)
- Absorción: vector (unidades/mes o %/mes, pero definir 1 unidad estándar)
- Output: TIR resultante por combinación.

07.03 Sanity checks (mínimos)
- Si sube Precio y todo lo demás constante, TIR debería subir (en general).
- Si sube Absorción (ventas más rápidas), TIR debería subir (por timing).
- Si el proyecto no tiene inversión inicial (error), TIR puede volverse absurda: detectar.

07.04 Spot checks (manual)
- Tomar 2 combinaciones y calcular el flujo y TIR por método alternativo (Excel/otra función) para validar.

08.00_SYNC MULTI-IA (CÓMO NO PERDER CONTROL)
