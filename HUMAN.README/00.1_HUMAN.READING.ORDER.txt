00.10_PROPOSITO_DEL_DOCUMENTO
======================================================================

Este documento define la jerarquía completa de lectura de la carpeta HUMAN.README.

Este archivo no contiene la filosofía del sistema, ni las reglas operativas, ni los
procedimientos técnicos del sistema HIA.

Este documento tiene un único objetivo:

ORGANIZAR LA LECTURA HUMANA DEL SISTEMA.

El propósito principal es evitar que el lector humano enfrente los documentos
de la carpeta HUMAN.README sin saber:

- qué archivo leer primero
- qué archivo leer después
- qué archivo leer solo si necesita profundizar
- qué archivo corresponde a operación
- qué archivo corresponde a conceptos
- qué archivo corresponde a resolución de problemas
- qué archivo corresponde a aprendizaje avanzado

Este documento establece una PRELACIÓN.

La prelación significa jerarquía de lectura.

La prelación significa orden lógico.

La prelación significa secuencia pedagógica.

Este documento es una guía de lectura.

Este documento no reemplaza ningún archivo.

Este documento no modifica ningún archivo.

Este documento no resume ningún archivo.

Este documento no sintetiza ningún archivo.

Este documento únicamente indica el ORDEN RECOMENDADO PARA LEER.

Este documento está escrito deliberadamente de forma extensa, didáctica,
académica y repetitiva para asegurar que el lector comprenda perfectamente
cómo navegar la carpeta HUMAN.README.

======================================================================
00.20_PRINCIPIO_FUNDAMENTAL_DE_LECTURA
======================================================================

La carpeta HUMAN.README contiene múltiples documentos.

Cada documento tiene un propósito distinto.

Algunos documentos están diseñados para:

- aprendizaje inicial
- comprensión conceptual
- operación del sistema
- control del sistema
- resolución de problemas
- documentación de casos reales

Si un lector abre documentos de forma aleatoria, es altamente probable que:

- no comprenda el contexto
- no entienda el propósito del documento
- interprete mal las instrucciones
- confunda conceptos con operaciones

Por esta razón, se define un ORDEN DE LECTURA.

El orden de lectura no es arbitrario.

El orden de lectura está diseñado para:

1) introducir al lector al sistema
2) permitir el primer uso del sistema
3) explicar cómo funciona el sistema
4) enseñar cómo trabajar correctamente con el sistema
5) explicar cómo observar y controlar el sistema
6) explicar cómo resolver problemas
7) presentar ejemplos reales de uso

Este documento debe ser leído antes de comenzar a explorar la carpeta HUMAN.README.

Este documento es el MAPA DE NAVEGACIÓN.

02.00_NIVEL_1_PRIMER_CONTACTO_CON_EL_SISTEMA
======================================================================

DOCUMENTO PRINCIPAL

09.0_HUMAN.START.RITUAL.txt

Este documento es el manual mínimo del sistema HIA.

Este documento está escrito específicamente para el usuario denominado
PEATÓN.

El término PEATÓN se utiliza para referirse a un usuario humano que no
necesariamente conoce la arquitectura del sistema, ni la filosofía completa
del sistema, ni las herramientas técnicas del sistema.

El documento START_RITUAL está diseñado para permitir que el usuario:

- ejecute HIA por primera vez
- entienda el flujo básico de trabajo
- interactúe con la IA de forma disciplinada
- ejecute tareas simples dentro del sistema

Este documento es intencionalmente operativo.

Este documento es práctico.

Este documento no intenta explicar todo el sistema.

Este documento no intenta explicar toda la arquitectura.

Este documento no intenta explicar todos los conceptos.

El objetivo de START_RITUAL es permitir que el usuario USE HIA.

El objetivo no es que el usuario comprenda completamente HIA.
