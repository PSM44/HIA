02.00_PACKAGES_POR_PHASE_(PHASE0..PHASE3.2)
==========

02.01 Regla base (aplica a todas las phases)

- Default IncludeRadar=None (cero RADAR dentro del paquete).
- Radar se agrega solo por toggle manual (Index / IndexLite).

02.02 Radar toggle policy (DEFAULT NONE)
- IncludeRadar=None (default):
  - no incluir outputs RADAR en el paquete.
- IncludeRadar=Index (toggle manual):
  - incluir: HIA_RAD_INDEX.REPO.ACTIVE.txt
- IncludeRadar=IndexLite (toggle manual):
  - incluir: HIA_RAD_INDEX.REPO.ACTIVE.txt
  - incluir: HIA_RAD_0001_LITE.ACTIVE.txt
- Prohibido por defecto: RADAR_FULL (solo forensic con orden explícita).

==========

- Los HUMAN en <PROJECT_ROOT>\HUMAN.README\ son la fuente.

03.30 Lock de edición (opcional)
- Si quieres “bloquear” DragnDrop para evitar edición manual:
  - Opción A: atributo ReadOnly en archivos
  - Opción B: test falla si detecta cambios manuales (hash mismatch vs fuente)
- Recomendación: empezar sin bloqueo duro, solo “regenerar siempre + test”.

06.00_GIT_POLICY_(DRAGNDROP_ES_RELEASE)
06.10 Decisión canónica
- DragnDrop\PhaseX\ se versiona en git como RELEASE ARTIFACT (sí se commitea).

06.20 Reglas (NO negociables)
- DragnDrop es GENERATED-ONLY:
  - Prohibido editar manualmente cualquier archivo bajo DragnDrop\.
  - El único modo válido de cambio es: ejecutar trigger correspondiente y regenerar.

06.30 Qué significa “release”
Un release DragnDrop (por phase) se considera válido solo si:
1) Build + Test PASS (trigger PhaseX),
2) README.txt regenerado,
3) DragnDrop\PhaseX\ contiene los required files del tester,
4) Commit + push posterior a PASS.

06.40 Qué se commitea
- Se commitea:
  - DragnDrop\Phase0\
  - DragnDrop\Phase1\
  - DragnDrop\Phase2\
  - DragnDrop\Phase3.1\
  - DragnDrop\Phase3.2\
- Se commitea el README generado (es parte del contrato y evidencia).

06.50 Qué NO se commitea (si existe)
- No se commitean outputs temporales (logs de ejecución) salvo que se declare explícitamente un “forensic checkpoint”.

06.60 Motivación
- DragnDrop en git habilita:
  - trazabilidad/auditoría
  - reproducibilidad del paquete adjuntado a IA cloud
  - rollback rápido por commit hash
