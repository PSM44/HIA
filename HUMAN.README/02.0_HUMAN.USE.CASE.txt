01.00_PROPOSITO_DEL_DOCUMENTO
======================================================================

Este documento describe casos reales de uso del sistema HIA.

El objetivo es mostrar cómo el sistema HIA se utiliza para resolver
problemas reales mediante la interacción estructurada entre humano
e inteligencia artificial.

Muchos documentos dentro de HUMAN.README explican reglas,
procedimientos y arquitectura del sistema.

Sin embargo, para comprender realmente el sistema es necesario
observar cómo se utiliza en situaciones reales.

Este documento proporciona ejemplos detallados de uso.

Cada caso de uso incluye:

contexto
objetivo
inputs
proceso
outputs
criterios de éxito

03.00_USE_CASE_001
======================================================================

IDENTIFICADOR_DEL_CASO

HIA.USECASE.001

DESCRIPCION_DEL_PROBLEMA

Evaluación financiera preliminar de un proyecto inmobiliario.

Se desea analizar si un proyecto inmobiliario es viable desde el
punto de vista financiero antes de invertir tiempo en un modelo
financiero detallado.

OBJETIVO_DEL_ANALISIS

Determinar rápidamente si el proyecto tiene potencial económico.

INPUTS

Precio del terreno
Superficie del terreno
Normativa urbanística
Costo estimado de construcción
Precio estimado de venta
Velocidad de absorción esperada

PROCESO

El humano entrega los datos iniciales a la IA.

La IA genera una estructura básica de modelo financiero.

Se calcula:

Ingresos estimados
Costos estimados
Margen bruto
TIR preliminar
VPN preliminar

OUTPUTS

Evaluación preliminar del proyecto.

El resultado puede indicar:

proyecto viable
proyecto marginal
proyecto no viable

CRITERIOS_DE_EXITO

El sistema permite generar una evaluación preliminar en menos de
una sesión de trabajo.

04.00_USE_CASE_002
======================================================================

IDENTIFICADOR_DEL_CASO

HIA.USECASE.002

DESCRIPCION_DEL_PROBLEMA

Análisis de documentación técnica.

Un humano necesita revisar documentos técnicos extensos y
extraer los elementos más relevantes.

OBJETIVO_DEL_ANALISIS

Reducir el tiempo necesario para comprender documentos complejos.

INPUTS

Documento técnico
Contexto del proyecto
Preguntas específicas

PROCESO

El documento se entrega a la IA.

La IA analiza el documento y responde preguntas específicas.

La IA identifica:

conceptos clave
riesgos
información crítica

OUTPUTS

Resumen estructurado del documento.

Lista de elementos críticos identificados.

CRITERIOS_DE_EXITO

El humano obtiene comprensión del documento sin necesidad de
leer completamente todo el contenido.

05.00_USE_CASE_003
======================================================================

IDENTIFICADOR_DEL_CASO

HIA.USECASE.003

DESCRIPCION_DEL_PROBLEMA

Diseño de arquitectura de software.

OBJETIVO_DEL_ANALISIS

Definir la arquitectura de un sistema antes de comenzar su
implementación.

INPUTS

Descripción del problema
Requerimientos del sistema
Restricciones técnicas

PROCESO

El humano describe el problema a la IA.

La IA propone una arquitectura inicial.

El humano revisa la propuesta y solicita mejoras.

Este proceso se repite hasta obtener una arquitectura satisfactoria.

OUTPUTS

Documento de arquitectura del sistema.

CRITERIOS_DE_EXITO

Se obtiene una arquitectura clara antes de comenzar el desarrollo.
