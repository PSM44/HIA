01.00_PROPOSITO_Y_MENTAL_MODEL
==========

01.10_PROPOSITO
- Este archivo existe por una razón práctica:
  cuando tienes muchos archivos HUMAN, cualquier cambio se vuelve engorroso si exige editar 10 copias divergentes.
- MASTER define un sistema para:
  - mantener ultra detalle en HUMAN,
  - sin edición manual multiarchivo,
  - con una única fuente de verdad por regla,
  - y sincronización automática FULL-copy.

01.20_MENTAL_MODEL
- Cada dominio de reglas tiene un “archivo dueño”.
- Ese archivo dueño contiene el CANON.
- Otros archivos pueden incluir el mismo contenido como DERIVED (FULL-copy) por usabilidad.
- Los derivados se actualizan por SYNC.RUNNER usando SYNC.MANIFEST.
- Git + RADAR + BATON entregan evidencia y trazabilidad.

==========
02.00_PRINCIPIOS_META_(REGLAS_DE_ORO)
==========

02.10_SINGLE-EDIT_RULE (Regla de edición única)
- Una regla o plantilla se edita en un solo lugar canónico (archivo dueño del dominio).
- Nadie edita manualmente copias en otros archivos.

02.20_DERIVED_FULL-COPY_EN_HUMAN
- En HUMAN, los derivados NO son resúmenes.
- Son FULL-copy: réplica completa del texto canónico.
- Esto cumple la exigencia de documentación académica ultra detallada.

02.30_TODO_CAMBIO_TIENE_EVIDENCIA
- Si se sincroniza contenido derivado:
  - se registra log,
  - se valida (mínimo),
  - y se hace commit (cuando corresponda).

02.40_CONFLICTOS_SE_RESUELVEN_POR_AUTORIDAD
- Si hay conflicto: manda el archivo dueño del dominio.
- MASTER es dueño solo de meta-gobernanza, no “de todo”.
